Code by Qwen3-Max & ChatGPT 5.2, 本人不负责任何模拟出现的错误。个人测试下来基本没问题。

pull.py为无捕获明光版本，radiance.py为有捕获明光版本

可以在main函数里修改相关初始参数，默认星辉全部立即换抽，氪金默认没石头就氪648=8080

radiance.jpg提供了详细的捕获明光机制，可以查看自己的抽卡记录来确定处于马尔可夫链的哪个位置。

做的比较粗糙的是对于抽到非up4星角色的处理，需要自己估一个抽到已满命的概率，不过这个也差不了太多。
